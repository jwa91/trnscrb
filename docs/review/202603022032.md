**Highest-Risk Findings**

- `P1` Deleting an active job only removes it from the UI; it does not cancel the underlying work, so uploads/transcription/delivery continue invisibly in the background. The delete affordance is exposed for active rows in [JobListView.swift:14](/Users/jw/developer/trnscrb/trnscrb/Presentation/Popover/JobListView.swift:14), deletion is a plain array removal in [JobListViewModel.swift:96](/Users/jw/developer/trnscrb/trnscrb/Presentation/ViewModels/JobListViewModel.swift:96), and the pipeline continues independently in [JobListViewModel.swift:147](/Users/jw/developer/trnscrb/trnscrb/Presentation/ViewModels/JobListViewModel.swift:147).
- `P1` The advertised “up to 1 GB / 3 hours” audio flow is not credible with the current uploader because the entire file is loaded into memory before upload. That is a straightforward memory-risk path in [S3Client.swift:38](/Users/jw/developer/trnscrb/trnscrb/Infrastructure/Storage/S3Client.swift:38).
- `P1` Unsupported file drops are silently ignored instead of surfacing the spec’d rejection/notification. The behavior is explicit in [JobListViewModel.swift:58](/Users/jw/developer/trnscrb/trnscrb/Presentation/ViewModels/JobListViewModel.swift:58) and is locked in by tests in [JobListViewModelTests.swift:42](/Users/jw/developer/trnscrb/Tests/Presentation/JobListViewModelTests.swift:42).

- `P1` Both drop handlers mutate shared arrays from asynchronous item-provider callbacks without synchronization; one path even uses `nonisolated(unsafe)` to suppress the compiler. That is a real concurrency hazard in [PopoverView.swift:90](/Users/jw/developer/trnscrb/trnscrb/Presentation/Popover/PopoverView.swift:90) and [DropZoneView.swift:60](/Users/jw/developer/trnscrb/trnscrb/Presentation/Popover/DropZoneView.swift:60).

- `P1` Retention cleanup is part of the documented MVP and exposed as a public use case, but calling it still crashes immediately via `fatalError` in [CleanupRetentionUseCase.swift:24](/Users/jw/developer/trnscrb/trnscrb/Domain/UseCases/CleanupRetentionUseCase.swift:24).
